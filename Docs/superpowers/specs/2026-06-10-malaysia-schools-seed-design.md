# Malaysian Secondary Schools Seed — Design Spec

**Date:** 2026-06-10  
**Status:** Approved  
**Author:** amir-mujahid

---

## Overview

A one-time Python script (`scripts/seed_schools.py`) queries the OpenStreetMap Overpass API,
filters results to Malaysian secondary schools, and writes a Flyway SQL migration file. The
migration replaces the V3 placeholder rows (some of which are primary schools) with ~2,000–3,000
real schools including GPS coordinates.

---

## Goals

- Replace placeholder V3 school rows with comprehensive real Malaysian secondary school data.
- Include GPS coordinates (lat/lng) for every school, enabling future geo-features.
- No runtime API dependency — data is baked into a committed Flyway migration file.
- Deterministic output so the migration file is stable across re-runs.

## Non-Goals

- Primary schools, kindergartens, polytechnics, universities.
- Automatic refresh / re-sync after the seed is committed.
- User-facing school management UI → Stage 7.

---

## Data Source

**OpenStreetMap Overpass API** — `https://overpass-api.de/api/interpreter`

- Free, no API key required.
- Query returns nodes, ways, and relations tagged `amenity=school` in Malaysia.
- Ways and relations include a `center` point, so every result has a usable lat/lng.
- OSM coverage for Malaysian schools: good for SMK/SMJK; MRSM and SBP boarding schools
  may be sparse (community-maintained data).

## School Type Filter

### Include (name starts with):

| Pattern | School type |
|---|---|
| `SMK ` | Sekolah Menengah Kebangsaan |
| `SMJK ` | Sekolah Menengah Jenis Kebangsaan |
| `SBP ` | Sekolah Berasrama Penuh |
| `MRSM ` | Maktab Rendah Sains MARA |
| `SM Sains` | Sekolah Menengah Sains |
| `SM Teknik` | Sekolah Menengah Teknik |
| `SM Agama` | Sekolah Menengah Agama |
| `SMA ` | Sekolah Menengah Agama (short form) |
| `Kolej Vokasional ` | Vocational college |
| `Sekolah Menengah` | Catch-all for name variants |
| `SM ` | General secondary school short-form |

### Exclude (skip if name starts with):

`SK `, `SRK`, `SRJK`, `SJK`, `Sekolah Rendah`, `Sekolah Kebangsaan ` (without Menengah),
`Tadika`, `Tabika`, `TABIKA`, `KEMAS`, `Taska`

Skip any entry with no `name` tag or no extractable coordinates.

## Script Design

**File:** `scripts/seed_schools.py`  
**Language:** Python 3.9+  
**Dependency:** `requests` (standard pip)

### Flow

```
1. Build Overpass QL query
   └─ [out:json][timeout:120]
      area["ISO3166-1"="MY"][admin_level=2]->.malaysia;
      nwr["amenity"="school"](area.malaysia);
      out center tags;

2. POST to https://overpass-api.de/api/interpreter
   (30–60 s; retry once on HTTP 429)

3. Parse JSON → list of elements

4. For each element:
   a. Extract name tag — skip if absent
   b. Extract lat/lng:
      - node: element["lat"] / element["lng"]
      - way / relation: element["center"]["lat"] / ["lon"]
   c. Apply include / exclude filter on lowercased name
   d. Add (name, lat, lng) to candidate list

5. Deduplicate:
   - Key = (name.strip(), round(lat, 4), round(lng, 4))
   - Keep first occurrence of each key

6. Sort by name (deterministic diff-friendly output)

7. Generate UUIDs deterministically:
   - uuid.uuid5(uuid.NAMESPACE_URL, f"isujana:school:{name}")
   - Guarantees the same school always gets the same UUID across re-runs

8. Write V7__seed_schools_malaysia.sql
```

### UUID strategy

Using `uuid5(NAMESPACE_URL, "isujana:school:{name}")` makes UUIDs stable across re-runs.
This means:
- Flyway checksum stays the same if the school list hasn't changed.
- Re-generating after adding new OSM data produces new rows for new schools only; existing
  UUIDs are unchanged so the migration remains forward-compatible.

## Migration Design

**File:** `backend/src/main/resources/db/migration/V7__seed_schools_malaysia.sql`

```sql
-- Generated by scripts/seed_schools.py on <date> — do not edit by hand.
-- Removes V3 placeholder rows that are primary or dummy schools,
-- then inserts real Malaysian secondary school data sourced from OSM.

DELETE FROM schools WHERE id IN (
    'a1b2c3d4-0000-0000-0000-000000000001',  -- SK Damansara Jaya  (primary)
    'a1b2c3d4-0000-0000-0000-000000000003',  -- SK Ara Damansara   (primary)
    'a1b2c3d4-0000-0000-0000-000000000004'   -- SK Taman Megah     (primary)
);

INSERT INTO schools (id, name, lat, lng) VALUES
    ('xxxxxxxx-xxxx-5xxx-...', 'SMK Subang Jaya',  3.0809, 101.5831),
    -- ~2,000–3,000 rows
ON CONFLICT (id) DO NOTHING;
```

- V3 rows for `SMK Subang Jaya` (`...000002`) and `SMK Tropicana` (`...000005`) are NOT
  deleted — they stay as valid secondary schools.
- The script skips inserting any school whose `uuid5` matches one of the kept V3 UUIDs
  (they use fixed UUIDs, so no collision is possible; `ON CONFLICT DO NOTHING` handles it).
- `tenant_id` is `NULL` for all seed rows. Stage 7 admin UI assigns schools to tenants.

## Running the Script (one-time)

```bash
# one-time setup
pip install requests

# run (30–60 s for Overpass query)
python scripts/seed_schools.py
```

Expected console output:
```
Querying Overpass API... (this may take 30-60s)
Fetched 4821 elements.
After name filter:   2743 secondary schools.
After dedup:         2718 schools.
Written: backend/src/main/resources/db/migration/V7__seed_schools_malaysia.sql
```

After generation:
1. Review the file (spot-check names, row count).
2. Commit it. **Once committed, do not re-generate.**

## Constraints & Caveats

| Constraint | Detail |
|---|---|
| OSM data quality | Community-maintained; MRSM/SBP may be under-represented |
| Re-run stability | uuid5 makes IDs deterministic; Flyway checksum stays stable |
| tenant_id | NULL on all seed rows — Stage 7 assigns schools to tenants |
| Overpass rate limit | Script uses `[timeout:120]` and retries once on HTTP 429 |
| No FK breakage | V3 kept rows (SMK Subang Jaya, SMK Tropicana) are preserved |
